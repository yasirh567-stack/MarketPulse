# Data Sources & Disclosures

Every price, news, sentiment, and event data point returned by the API carries a
`data_status` field: `live | delayed | historical | cached | demo`. The frontend
renders this as a badge everywhere data is shown. This document explains what each
source actually is.

## Market data — `yfinance`

- **What it is**: an open-source Python client for Yahoo Finance's public
  (unofficial) endpoints. Free, no API key.
- **Status labeling**: intraday quotes are tagged `delayed` (Yahoo's free data is
  typically ~15 minutes behind for US equities — there is no SLA). Multi-day
  history is tagged `historical`.
- **Limitations**: no official support contract, rate limits are informal and can
  change without notice, and coverage/accuracy for illiquid or non-US tickers can be
  inconsistent. The app catches provider failures and falls back to cached data (if
  any) or demo fixtures rather than crashing.
- **Why chosen over alternatives**: free, real-tier alternatives (Alpha Vantage,
  IEX Cloud, Polygon) all impose either a paid tier for meaningful call volume or
  much stricter free-tier rate limits than a demo dashboard needs.
- **Ticker/company search**: `yfinance` has no first-class search endpoint of its
  own, so the search box resolves company names (e.g. "netflix") to tickers via
  Yahoo Finance's free, public, unofficial search endpoint
  (`query1.finance.yahoo.com/v1/finance/search`) — same no-key posture as the
  quote/history calls, just a different endpoint. Non-equity quote types (ETFs,
  currencies, etc.) are filtered out. An exact ticker the search endpoint doesn't
  surface still resolves via a direct quote-validation fallback.

## News — Google News RSS

- **What it is**: a public RSS search feed (`news.google.com/rss/search`), queried
  per ticker. Free, no API key, no scraping of any page beyond the RSS XML itself
  (which is an explicitly public syndication format, not a page rendered for
  humans).
- **Status labeling**: tagged `cached` once stored — RSS delivery timing isn't
  guaranteed to be immediate, so it is never presented as a live newswire.
- **Limitations**: no guaranteed coverage of any specific publisher, headline
  quality/relevance varies, and Google can change or rate-limit this endpoint
  without notice. All retrieved titles/summaries are HTML-sanitized
  (`app/core/sanitize.py`) before storage or display.
- **Deduplication**: articles are deduplicated by a SHA-256 hash of their URL, so
  the same story surfaced by repeated queries collapses to one stored row.

## Social data — Reddit (optional, off by default)

- **What it is**: Reddit's free OAuth "script app" API, used read-only for a
  simple keyword search. Requires the user to create their own free Reddit app
  and set `ENABLE_REDDIT=true` plus `REDDIT_CLIENT_ID`/`REDDIT_CLIENT_SECRET`.
- **Why optional**: the platform must work with zero API keys out of the box;
  Reddit requires registering an app, so it's an enhancement, never a dependency.
  With it disabled, sentiment aggregation simply has zero social-sourced rows and
  relies entirely on news sentiment — nothing breaks.

## Macro valuation — FRED (optional, off by default)

- **What it is**: the Federal Reserve's free, public FRED API, used for the
  Buffett Indicator widget (Status page) — a market-cap-to-GDP valuation
  gauge. Requires a free API key
  (https://fred.stlouisfed.org/docs/api/api_key.html) and
  `ENABLE_FRED=true` + `FRED_API_KEY=...`.
- **Methodology**: approximated as the Wilshire 5000 Full Cap Price Index
  (FRED series `WILL5000INDFC`, a free proxy for total US market
  capitalization) divided by nominal GDP (FRED series `GDP`), following the
  commonly cited "Buffett Indicator" heuristic. This is a widely used
  do-it-yourself approximation, not an official aggregate market-cap figure —
  disclosed directly in the API response's `methodology_note`.
- **Why optional**: same reasoning as Reddit — requires a free key, so it's
  an enhancement. Disabled (or on any fetch failure), the widget falls back to
  a clearly-labeled synthetic demo series rather than breaking or guessing.

## Demo fixtures (`data/demo/`)

Used whenever `DEMO_MODE=true`, or as an automatic fallback if a live provider call
fails for a ticker that happens to also be one of the bundled demo tickers
(AAPL, MSFT, TSLA, NVDA, GME).

- **Price series**: generated by `scripts/generate_demo_fixtures.py` using a
  **seeded geometric Brownian motion** (pure Python, fixed seeds per ticker) — a
  synthetic stand-in for a price history, never real historical prices. Every bar
  is tagged `data_status: demo`.
- **News**: a small set of hand-written, realistic-sounding but entirely fictional
  headlines per ticker (dividend announcements, analyst actions, litigation, etc.),
  designed to exercise every event-detection category.
- **Why offsets, not fixed dates**: fixtures store *relative* offsets
  (`offset_trading_days` for prices, `hours_ago` for news) rather than baked-in
  calendar dates. The demo providers resolve these to real dates/timestamps
  relative to "now" at request time — so a recruiter running this demo years from
  now still sees a price history ending "today" and news from "a few hours ago,"
  without ever needing to regenerate the fixtures or risk stale-looking data.
- **Sentiment on demo text is still real**: VADER (or FinBERT, if enabled) actually
  scores the demo headlines live — the sentiment/prediction pipeline is not
  faked, only the underlying price/news text is synthetic.

## What is never done

- No scraping of any page in violation of its terms of service.
- No mixing of demo and real data within a single response — a ticker's data is
  either fully real (with the appropriate `live`/`delayed`/`historical`/`cached`
  label) or fully demo (`demo` label), never blended silently.
- No fabricated citations, prices, or headlines anywhere, including in the market
  assistant (see `docs/architecture.md`'s note on the TF-IDF retrieval assistant).
